연차 관리 (ADMIN)

관리자 전용 연차 승인/반려 페이지입니다. 승인 시 직원의 잔여 연차를 차감하고 캘린더에 일정을 자동 생성합니다.

마지막 업데이트 2026-05-14

📋 개요

  • → /admin/leave — PENDING 연차 목록
  • → /admin/leave/approve — 일괄 승인 화면
  • → /admin/leave/[id] — 단건 상세 + 승인/반려
🚨

ADMIN 전용: src/proxy.ts에서 /admin/* 경로에 EMPLOYEE가 접근하면 홈으로 리다이렉트합니다. API에서도 반드시 role을 재검증하세요.

export async function POST(req: Request, { params }: { params: { id: string } }) {
  const session = await auth()
  if (session?.user.role !== "ADMIN")
    return NextResponse.json({ error: "Forbidden" }, { status: 403 })

  const leave = await db.leaveRequest.findUnique({
    where: { id: params.id },
    include: { user: true },
  })
  if (!leave || leave.status !== "PENDING")
    return NextResponse.json({ error: "Not Found" }, { status: 404 })

  const cost = leave.type === "ANNUAL" ? 1 : 0.5

  await db.$transaction([
    db.leaveRequest.update({
      where: { id: params.id },
      data:  { status: "APPROVED" },
    }),
    db.user.update({
      where: { id: leave.userId },
      data:  { annualLeave: { decrement: cost } },
    }),
    db.schedule.create({
      data: {
        title:          `[연차] ${leave.user.email}`,
        startAt:        leave.date,
        endAt:          leave.date,
        allDay:         leave.type === "ANNUAL",
        creatorId:      session!.user.id,
        leaveRequestId: leave.id,
      },
    }),
    db.notification.create({
      data: {
        userId:  leave.userId,
        message: "연차 신청이 승인되었습니다.",
        url:     `/calendar/${leave.id}`,
      },
    }),
  ])

  return NextResponse.json({ ok: true })
}

🔒 보안

  • ✓ 모든 /api/admin/* 엔드포인트에서 ADMIN role 재검증 필수
  • ✓ 승인·annualLeave 차감·Schedule 생성을 Prisma $transaction으로 원자적 처리
  • ✓ 이미 APPROVED/REJECTED인 신청 재처리 방지 (status=PENDING 확인)
