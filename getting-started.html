시작하기

처음 프로젝트를 인계받은 개발자가 로컬 환경을 세팅하고 개발 서버를 실행하기까지의 전 과정을 설명합니다.

마지막 업데이트 2026-05-14

📋 사전 준비

개발을 시작하기 전에 아래 도구가 설치되어 있어야 합니다.

도구권장 버전확인 명령어비고
Node.jsv20 LTS 이상node -vv18 미만은 동작 보장 불가
npmv10 이상npm -vNode.js 설치 시 자동 포함
Git최신git --version
PostgreSQLv15 이상 (로컬) 또는 Neon/Supabasepsql --version로컬 DB 없으면 Neon 무료 플랜 사용 권장
💡

Node.js 버전 관리: nvm (Mac/Linux) 또는 fnm (Windows/Mac/Linux)을 사용해 Node 버전을 관리하면 버전 충돌을 방지할 수 있습니다. nvm use 20으로 프로젝트 버전으로 전환하세요.

⚠️

외부 서비스 API 키 발급 필요: Deepgram, Gemini, VAPID 키 등이 없으면 AI 회의록 및 웹 푸시 기능은 동작하지 않습니다. 외부 서비스 가이드를 먼저 확인하세요.

📥 클론 & 설치

BASH1. 레포지토리 클론
# GitHub 레포지토리 클론
git clone https://github.com/투게이트컴즈/togate-agent.git
cd togate-agent

# 현재 브랜치 확인
git branch -a
BASH2. 의존성 설치
npm install
⚠️

npm ci 권장: 팀 환경에서는 npm install 대신 npm ci를 사용하면 package-lock.json을 기준으로 정확한 버전이 설치됩니다.

⚙️ 환경변수 설정

프로젝트 루트에 .env.local 파일을 생성합니다. 실제 키 값은 외부 서비스 담당자에게 전달받으세요.

BASH.env.local 파일 생성
# 예시 파일 복사 후 값 채우기
cp .env.example .env.local
ENV.env.local — 전체 템플릿
# ── Auth.js ──────────────────────────────────────────
AUTH_SECRET=           # openssl rand -base64 33 으로 생성
AUTH_URL=http://localhost:3000   # 로컬 개발 시에만 설정
AUTH_TRUST_HOST=true

# ── Database ─────────────────────────────────────────
DATABASE_URL=postgresql://USER:PASSWORD@HOST:5432/togate

# ── AI (gemini | openai) ─────────────────────────────
AI_PROVIDER=gemini
GEMINI_API_KEY=
OPENAI_API_KEY=           # AI_PROVIDER=openai 시에만 필요

# ── 음성 전사 (Deepgram) ────────────────────────────
DEEPGRAM_API_KEY=
DEEPGRAM_PROJECT_ID=

# ── 웹 푸시 VAPID ────────────────────────────────────
# npx web-push generate-vapid-keys 로 생성
NEXT_PUBLIC_VAPID_PUBLIC_KEY=
VAPID_PRIVATE_KEY=
VAPID_MAILTO=mailto:admin@togate.com

# ── Cron ─────────────────────────────────────────────
CRON_SECRET=             # 로컬에서는 임의 문자열로 설정

필수 키 빠른 생성

BASHAUTH_SECRET 및 VAPID 키 생성
# AUTH_SECRET 생성
openssl rand -base64 33

# VAPID 키 쌍 생성 (public + private)
npx web-push generate-vapid-keys
🚨

.env.local은 절대 Git에 커밋하지 마세요. .gitignore에 이미 포함되어 있으나, 팀 내 키 공유는 반드시 별도 보안 채널(1Password, Notion 비공개 페이지 등)을 이용하세요.

🗄️ DB 초기화

로컬 PostgreSQL 또는 Neon 인스턴스에 스키마를 적용하고 초기 ADMIN 계정을 생성합니다.

BASHDB 마이그레이션 & 시드
# 1. 스키마 마이그레이션 적용
npm run db:migrate

# 2. 초기 ADMIN 계정 생성
npm run db:seed
#    → admin@togate.com / admin1234! 로 생성됨
#    → 첫 로그인 시 비밀번호 변경 강제됨

# 3. (선택) Prisma Studio로 DB 확인
npm run db:studio
⚠️

운영 DB에 db:migrate 실행 금지: 운영 환경에서는 prisma migrate dev 대신 prisma migrate deploy를 사용합니다. migrate dev는 개발 전용이며 운영 데이터에 영향을 줄 수 있습니다.

명령어환경설명
npm run db:migrate개발 전용prisma migrate dev — 마이그레이션 파일 생성 + 적용
prisma migrate deploy운영/스테이징기존 마이그레이션 파일만 순서대로 적용 (안전)
npm run db:seed최초 1회초기 ADMIN 계정 admin@togate.com 생성
npm run db:studio개발 전용Prisma Studio (DB GUI) — localhost:5555

☑️ 온보딩 체크리스트

✅

아래 항목을 모두 완료하면 개발 준비가 끝납니다.

환경 설정

  • ✓ Node.js v20 이상 설치 확인 (node -v)
  • ✓ 레포지토리 클론 완료
  • ✓ npm install 완료 (오류 없음)
  • ✓ .env.local 파일 생성 및 모든 필수 키 입력
  • ✓ npm run db:migrate 완료 (마이그레이션 오류 없음)
  • ✓ npm run db:seed 완료 (admin 계정 생성)
  • ✓ npm run dev 실행 후 localhost:3000 접속 확인

기능 동작 확인

  • ✓ admin@togate.com 로그인 → 비밀번호 변경 강제 이동 확인
  • ✓ 공지사항 작성 → 푸시 알림 발송 확인
  • ✓ AI 회의록 — 음성 녹음 → 전사 결과 확인 (Deepgram 키 필요)
  • ✓ 연차 신청 → ADMIN 승인 플로우 확인

접근 권한 확인

  • ✓ GitHub 레포지토리 접근 권한 부여받음
  • ✓ Vercel 프로젝트 팀 초대 수락 (외부 서비스 가이드 참고)
  • ✓ 외부 서비스 API 키 전달받음 (Deepgram, Gemini, VAPID)
