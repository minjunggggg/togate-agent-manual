일정 관리

팀 일정 CRUD 기능입니다. 참석자 초대, 종일 이벤트, 연차 연동을 지원합니다. 매일 00:00 UTC Cron으로 당일 일정 알림을 발송합니다.

마지막 업데이트 2026-05-14

📋 개요

  • → /schedule — 일정 목록
  • → /schedule/week — 주간 뷰
  • → /schedule/new — 일정 생성
  • → /schedule/[id] — 상세
  • → /schedule/[id]/edit — 수정
💡

연차 연동: Schedule.leaveRequestId로 연차신청과 연결됩니다. 연차 승인 시 해당 날짜에 자동으로 Schedule이 생성됩니다.

TSapp/api/schedules/route.ts — POST
export async function POST(req: Request) {
  const session = await auth()
  if (!session?.user)
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const { title, startAt, endAt, allDay, inviteeIds } = await req.json()

  const schedule = await db.schedule.create({
    data: {
      title, startAt, endAt, allDay: allDay ?? false,
      inviteeIds: inviteeIds ?? [],
      creatorId: session.user.id,
    },
  })

  // 초대된 직원에게 앱 내 알림 생성
  if (inviteeIds?.length) {
    const users = await db.employee.findMany({
      where: { id: { in: inviteeIds } },
      select: { userId: true },
    })
    await db.notification.createMany({
      data: users.map(({ userId }) => ({
        userId,
        message: `일정에 초대되었습니다: ${title}`,
        url:     `/schedule/${schedule.id}`,
      })),
    })
  }

  return NextResponse.json(schedule, { status: 201 })
}

🗄️ DB 모델

TSprisma/schema.prisma — Schedule
model Schedule {
  id              String   @id @default(cuid())
  title           String
  startAt         DateTime
  endAt           DateTime
  allDay          Boolean  @default(false)
  inviteeIds      String[]          // Employee ID 배열
  creatorId       String
  leaveRequestId  String?           // 연차 승인 시 연결
  creator         User     @relation(fields: [creatorId], references: [id])
  leaveRequest    LeaveRequest? @relation(fields: [leaveRequestId], references: [id])
}
💡

inviteeIds 타입: PostgreSQL의 String[]로 Employee ID를 저장합니다. 참석자 이름을 렌더링할 때는 employee.findMany({ where: { id: { in: inviteeIds } } })로 별도 조회해야 합니다.

⏰ Cron — 당일 일정 알림

매일 00:00 UTC(한국 오전 9시)에 당일 일정을 조회해 초대된 직원에게 앱 내 알림을 발송합니다.

TSvercel.json — cron 설정
{
  "crons": [
    { "path": "/api/cron/daily-schedule-reminder", "schedule": "0 0 * * *"  },
    { "path": "/api/cron/reset-annual-leave",         "schedule": "0 15 31 12 *" }
  ]
}
⚠️

Cron 인증: Vercel Cron은 Authorization: Bearer 헤더로 호출합니다. Route Handler에서 CRON_SECRET을 검증해 외부 호출을 차단하세요.
