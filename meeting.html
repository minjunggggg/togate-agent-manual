AI 회의록

음성을 녹음·업로드하면 AI가 자동으로 회의록을 생성합니다. 기본 모델은 gemini-2.5-flash이며, AI_PROVIDER=openai로 gpt-4o-mini로 전환할 수 있습니다.

마지막 업데이트 2026-05-14

📋 개요

  • → /meeting — 회의록 목록 (폴더 단위)
  • → /meeting/new — 새 회의록 시작 (녹음 vs 파일 업로드 선택)
  • → /meeting/new/record — 실시간 음성 녹음
  • → /meeting/new/result — AI 처리 결과 확인 및 저장
  • → /meeting/[id] — 회의록 상세
  • → /meeting/folder/[id] — 폴더 내 목록

TSapp/api/meetings/transcribe/route.ts
import { getAIProvider } from "@/lib/ai/provider"

export async function POST(req: Request) {
  const session = await auth()
  if (!session?.user)
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const formData = await req.formData()
  const audioFile = formData.get("audio") as File
  if (!audioFile)
    return NextResponse.json({ error: "audio required" }, { status: 400 })

  const ai = getAIProvider()
  const result = await ai.transcribeAndSummarize(audioFile)

  return NextResponse.json({
    transcript:  result.transcript,
    summary:     result.summary,
    keywords:    result.keywords,   // string[]
    speakerMap:  result.speakerMap,  // { [speaker]: string[] }
  })
}

🤖 AI 프로바이더 추상화

src/lib/ai/provider.ts가 환경변수에 따라 Gemini 또는 OpenAI를 선택합니다. 코드 내에서 직접 모델을 지정하지 마세요.

TSsrc/lib/ai/provider.ts
type AIProvider = {
  transcribeAndSummarize(audio: File): Promise<{
    transcript: string
    summary:    string
    keywords:   string[]
    speakerMap: Record<string, string[]>
  }>
}

export function getAIProvider(): AIProvider {
  if (process.env.AI_PROVIDER === "openai") {
    return new OpenAIProvider()  // gpt-4o-mini
  }
  return new GeminiProvider()    // gemini-2.5-flash (기본)
}
AI_PROVIDER모델특징
(미설정, 기본)gemini-2.5-flash멀티모달 음성 직접 처리, 긴 컨텍스트
openaigpt-4o-miniWhisper STT → GPT 요약 2-step
⚠️

모델 변경 시: AI_PROVIDER만 바꾸면 됩니다. provider.ts를 직접 수정하지 마세요. 두 프로바이더의 출력 형식(keywords, speakerMap)은 동일하게 정규화되어야 합니다.

🗄️ DB 모델

TSprisma/schema.prisma — MeetingMinute · MeetingFolder
model MeetingMinute {
  id         String        @id @default(cuid())
  title      String
  transcript String        // 전체 텍스트
  summary    String        // AI 요약
  keywords   String[]      // 핵심 키워드 배열
  speakerMap Json          // { "화자A": ["발언1", ...] }
  folderId   String?
  creatorId  String
  createdAt  DateTime      @default(now())
  folder     MeetingFolder? @relation(fields: [folderId], references: [id])
  creator    User           @relation(fields: [creatorId], references: [id])
}

model MeetingFolder {
  id        String          @id @default(cuid())
  name      String
  type      FolderType      // PERSONAL | SHARED
  ownerId   String
  deletedAt DateTime?       // 소프트 딜리트
  minutes   MeetingMinute[]
  owner     User            @relation(fields: [ownerId], references: [id])
}

enum FolderType { PERSONAL  SHARED }
💡

speakerMap 타입: Prisma의 Json 타입으로 저장됩니다. 클라이언트에서 읽을 때 as Record<string, string[]>로 캐스팅하세요.

📂 폴더 관리

타입접근설명
PERSONAL본인만개인 회의록 보관함
SHARED모든 직원팀 공유 회의록 폴더
⚠️

PERSONAL 폴더 접근 제어: API에서 type === "PERSONAL"인 경우 ownerId === session.user.id를 반드시 확인하세요. 타인의 개인 폴더에 접근하면 403을 반환합니다.
