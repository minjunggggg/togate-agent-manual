프로젝트 개요

Togate 사내앱의 기능, 기술 스택, 폴더 구조, 핵심 파일을 한눈에 파악합니다.

마지막 업데이트 2026-05-14

📋 프로젝트 소개

Togate-agent는 모바일 웹 기반 PWA 사내앱입니다. AI 음성 회의록, 일정/연차 관리, 공지사항, 비상연락망, 웹 푸시 알림 기능을 포함하며 Capacitor 8을 통해 iOS 네이티브 앱으로도 배포됩니다.

💡

사내 전용 앱: 공개 회원가입이 없습니다. 모든 계정은 ADMIN이 직접 생성하며, 첫 로그인 시 비밀번호 변경이 강제됩니다.

주요 기능

  • → AI 음성 회의록 (Gemini 2.5 Flash / GPT-4o-mini)
  • → 일정 관리 + 연차/반차 신청 및 반려
  • → 공지사항 (전직원 작성, 푸시 알림)
  • → 비상연락망 (직원 디렉토리)
  • → 웹 푸시 알림 (Service Worker + VAPID)
  • → Capacitor 8 iOS 네이티브 앱 래퍼

🚨

🗂️ 핵심 파일

파일 역할
src/proxy.ts 인증 게이트, 초기 비밀번호 강제 변경, admin 권한 체크. Next.js 16에서 middleware.ts 대체
src/lib/db.ts Prisma 클라이언트 싱글턴. globalThis로 HMR 중복 생성 방지. export const prisma로 사용
src/lib/auth.ts NextAuth(Auth.js v5) 설정. Credentials Provider, JWT 세션, 커스텀 로그인 페이지
src/lib/ai/provider.ts AI 추상화 레이어. AIProvider()를 호출하면 AI_PROVIDER env에 따라 Gemini 또는 OpenAI 반환
src/lib/push.ts 웹 푸시 발송. sendPushToUser(userId, payload) — 특정 유저의 모든 구독에 병렬 발송. VAPID 설정은 첫 호출 시 1회만 초기화. 만료된 구독(410)은 자동 삭제
prisma/schema.prisma DB 스키마. generator output은 반드시 ../src/generated/prisma로 설정
prisma.config.ts Prisma 7 설정 파일. 스키마·마이그레이션 경로, 시드 명령어(npx tsx prisma/seed.ts), DATABASE_URL datasource URL을 한 곳에서 관리

Prisma 7 설정 규칙

Prismaprisma/schema.prisma — generator 설정
generator client {
  provider = "prisma-client"
  output   = "../src/generated/prisma"
}

// datasource에 url = env(...) 작성 불필요 — prisma.config.ts가 처리
datasource db {
  provider = "postgresql"
}
TSsrc/lib/db.ts — Prisma 클라이언트 
import { PrismaClient } from "@/generated/prisma/client"
import { PrismaPg } from "@prisma/adapter-pg"

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

function createPrismaClient() {
  const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL! })
  return new PrismaClient({ adapter, log: ["error"] })
}

export const prisma = globalForPrisma.prisma ?? createPrismaClient()

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma

🗄️ DB 모델

모델 주요 필드 설명
User email, name, password, role, annualLeave, usedAnnualLeave, passwordChanged, lastLoginAt? role: ADMIN / EMPLOYEE. annualLeave 기본 15, usedAnnualLeave 기본 0
Employee name, email?, phone?, position?, job?, residence?, userId? @unique User와 1:1 연결 (optional). 비상연락망에서 사용
LeaveRequest type, status, startDate, endDate, days, reason, visibility, userId, approverId?, approvedAt? type: ANNUAL(1일) / HALF_DAY_AM / HALF_DAY_PM(0.5일). status: PENDING / APPROVED / REJECTED. visibility 기본 "PRIVATE"
Schedule title, description?, startDate, endDate, allDay, inviteeIds String[], leaveRequestId?, createdById inviteeIds는 Employee ID 배열. 연차 승인 시 leaveRequestId 연결
MeetingMinute title, content, rawTranscript?, attendees String[], keywords String[], speakerMap Json?, meetingDate, folderId?, createdById AI 회의록. speakerMap은 Record<string, string[]>로 캐스팅
MeetingFolder name, type, createdById, deletedAt? type: PERSONAL / SHARED. 소프트 딜리트 (deletedAt). PERSONAL은 createdById 일치 검증 필수
Notice title, content, imageUrl?, createdById 공지사항. 작성 후 웹 푸시 발송
Notification type, title, message, referenceId?, isRead, userId 앱 내 알림. 일정 초대, 연차 승인/반려 시 생성
PushSubscription endpoint, p256dh, auth, userId 웹 푸시 구독 정보. 410 응답 시 자동 삭제

⏰ Vercel Cron

vercel.json에 정의된 두 개의 Cron Job이 있습니다.

경로 스케줄 동작
/api/cron/daily-schedule-reminder 0 0 * * * (매일 00:00 UTC) 당일 일정이 있는 직원에게 알림 발송
/api/cron/reset-annual-leave 0 15 31 12 * (12/31 15:00 UTC = 1/1 00:00 KST) 전 직원 annualLeave를 15일로 초기화
TSCron 인증 — Authorization 헤더 검증 
export async function GET(req: Request) {
  const authHeader = req.headers.get("authorization")
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return Response.json({ error: "Unauthorized" }, { status: 401 })
  }
  // ... cron logic
}
⚠️

CRON_SECRET: Vercel 환경변수에 설정해야 합니다. Cron Job 요청 시 Vercel이 자동으로 Authorization: Bearer {CRON_SECRET} 헤더를 추가합니다.

⚙️ 환경변수

프로젝트 루트의 .env.local 파일과 Vercel 환경변수에 아래 값을 설정하세요.

ENV.env.local — 전체 환경변수
# Auth.js (NextAuth v5)
AUTH_SECRET=your-32-char-random-secret
AUTH_URL=http://localhost:3000
AUTH_TRUST_HOST=true

# Database (Prisma 7 + PostgreSQL)
DATABASE_URL=postgresql://user:pass@localhost:5432/togate

# AI (gemini | openai)
AI_PROVIDER=gemini
GEMINI_API_KEY=your-gemini-api-key
OPENAI_API_KEY=your-openai-api-key

# 음성 전사 (Deepgram)
DEEPGRAM_API_KEY=your-deepgram-api-key
DEEPGRAM_PROJECT_ID=your-deepgram-project-id

# 웹 푸시 VAPID
NEXT_PUBLIC_VAPID_PUBLIC_KEY=your-vapid-public-key

# Vercel
VERCEL_OIDC_TOKEN=# Vercel이 자동 주입 — 직접 설정 불필요

# Cron
CRON_SECRET=your-cron-secret
변수 필수 설명
AUTH_SECRET 필수 JWT 서명·암호화 키. 32자 이상 랜덤 문자열. openssl rand -base64 33으로 생성
AUTH_URL 로컬만 Auth.js가 콜백 URL을 구성할 때 사용하는 기준 URL. Vercel 배포 시 자동 주입됨 — 직접 설정 시 Preview URL 충돌 주의
AUTH_TRUST_HOST 필수 Vercel·역방향 프록시 환경에서 true로 설정. Host 헤더를 신뢰해 리다이렉트 URL을 올바르게 구성함
DATABASE_URL 필수 PostgreSQL 연결 문자열. Prisma 7 + @prisma/adapter-pg가 사용. prisma.config.ts가 읽음
AI_PROVIDER 선택 gemini(기본) 또는 openai. src/lib/ai/provider.ts의 AIProvider()가 이 값을 읽어 클라이언트를 반환
GEMINI_API_KEY 필수 Google AI Studio에서 발급. AI_PROVIDER=gemini(기본)일 때 회의록 전사·요약에 사용
OPENAI_API_KEY 선택 OpenAI API 키. AI_PROVIDER=openai로 전환 시 필요. GPT-4o-mini 모델 사용
VERCEL_OIDC_TOKEN 자동 Vercel이 배포 시 자동으로 주입하는 OIDC 토큰. 직접 설정 불필요. 외부 서비스 인증(예: GCP Workload Identity)에 활용 가능
DEEPGRAM_API_KEY 필수 Deepgram 음성 전사 API 키. 회의록 녹음 파일을 텍스트로 변환하는 데 사용
DEEPGRAM_PROJECT_ID 필수 Deepgram 프로젝트 ID. API 키와 함께 사용해 프로젝트별 사용량·설정을 관리
NEXT_PUBLIC_VAPID_PUBLIC_KEY 필수 VAPID 공개 키. NEXT_PUBLIC_ 접두사로 브라우저에 노출됨. Service Worker가 구독 등록 시 사용
VAPID_PRIVATE_KEY 필수 VAPID 비공개 키. 서버에서만 사용. src/lib/push.ts의 푸시 발송 시 서명에 사용
VAPID_MAILTO 필수 VAPID 연락처. mailto: 형식으로 작성. 푸시 서비스가 문제 발생 시 연락하는 이메일
CRON_SECRET 필수 Vercel Cron이 API Route 호출 시 자동으로 Authorization: Bearer {CRON_SECRET} 헤더를 추가. Cron Route Handler에서 검증
💡

VAPID 키 생성: npx web-push generate-vapid-keys를 실행하면 공개 키와 비공개 키가 한 번에 출력됩니다.

💡

Vercel 배포 팁: AUTH_URL은 Vercel Preview 환경에서 URL 충돌을 일으킬 수 있으므로 설정하지 않는 것이 권장됩니다. AUTH_TRUST_HOST=true만으로 충분합니다.

💻 개발 명령어

Shell개발 명령어
# 개발 서버 (webpack 빌더 사용)
npm run dev          # next dev --webpack

# DB 관리
npm run db:migrate   # prisma migrate dev
npm run db:seed      # 초기 관리자 계정 생성
npm run db:studio    # Prisma Studio 실행

# 빌드 (Prisma 생성 포함)
npm run build        # prisma generate && next build --webpack
명령어 설명
npm run dev next dev --webpack — webpack 빌더로 개발 서버 실행
npm run db:migrate prisma migrate dev — DB 스키마 마이그레이션
npm run db:seed 초기 관리자 계정 생성 — admin@togate.com / admin1234!
npm run db:studio Prisma Studio (DB GUI) 실행
npm run build prisma generate && next build --webpack — Prisma 클라이언트 생성 후 빌드
