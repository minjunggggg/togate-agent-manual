프로젝트 개요

Togate 사내앱의 기능, 기술 스택, 폴더 구조, 핵심 파일을 한눈에 파악합니다.

마지막 업데이트 2026-05-14

📋 프로젝트 소개

Togate는 모바일 웹 기반 PWA 사내앱입니다. AI 음성 회의록, 일정/연차 관리, 공지사항, 비상연락망, 웹 푸시 알림 기능을 포함하며 Capacitor 8을 통해 iOS 네이티브 앱으로도 배포됩니다.

💡

사내 전용 앱: 공개 회원가입이 없습니다. 모든 계정은 ADMIN이 직접 생성하며, 첫 로그인 시 비밀번호 변경이 강제됩니다.

주요 기능

  • → AI 음성 회의록 (Gemini 2.5 Flash / GPT-4o-mini)
  • → 일정 관리 + 연차/반차 신청 및 반려
  • → 공지사항 (전직원 작성, 푸시 알림)
  • → 비상연락망 (직원 디렉토리)
  • → 웹 푸시 알림 (Service Worker + VAPID)
  • → Capacitor 8 iOS 네이티브 앱 래퍼

🚨

Prisma 7 import 경로: @/generated/prisma/client — @prisma/client가 아닙니다.

🗂️ 핵심 파일

파일 역할
src/proxy.ts 인증 게이트, 초기 비밀번호 강제 변경, admin 권한 체크. Next.js 16에서 middleware.ts 대체
src/lib/db.ts Prisma 클라이언트 싱글턴. PrismaPg 어댑터 주입 필수
src/lib/auth.ts NextAuth(Auth.js v5) 설정. Credentials Provider, JWT 세션, 커스텀 로그인 페이지
src/lib/ai/provider.ts AI 추상화 레이어. getAIProvider()를 호출하면 AI_PROVIDER env에 따라 Gemini 또는 OpenAI 반환
src/lib/push.ts 웹 푸시 발송. sendPushToAll() — 만료된 구독(410/404)은 자동 삭제
prisma/schema.prisma DB 스키마. generator output은 반드시 ../src/generated/prisma로 설정
prisma.config.ts Prisma 7 설정 파일. DATABASE_URL 등 datasource 설정 처리

Prisma 7 설정 규칙

Prismaprisma/schema.prisma — generator 설정
generator client {
  provider = "prisma-client"
  output   = "../src/generated/prisma"
}

// datasource에 url = env(...) 작성 불필요 — prisma.config.ts가 처리
datasource db {
  provider = "postgresql"
}
TSsrc/lib/db.ts — Prisma 클라이언트 
import { PrismaPg } from "@prisma/adapter-pg"
import { PrismaClient } from "@/generated/prisma/client"

const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL })
export const db = new PrismaClient({ adapter })

🗄️ DB 모델

모델 주요 필드 설명
User email, hashedPassword, role, annualLeave, passwordChanged role: ADMIN / EMPLOYEE. 기본 annualLeave = 15
Employee name, phone?, position, userId @unique User와 1:1 연결. 비상연락망에서 사용
LeaveRequest type, status, date, userId type: ANNUAL(1일) / HALF_DAY_AM / HALF_DAY_PM(0.5일). status: PENDING / APPROVED / REJECTED
Schedule inviteeIds String[], allDay, leaveRequestId? inviteeIds는 Employee ID 배열. 연차 승인 시 leaveRequestId 연결
MeetingMinute transcript, summary, keywords String[], speakerMap Json AI 회의록. speakerMap은 Record<string, string[]>로 캐스팅
MeetingFolder type, ownerId?, deletedAt? type: PERSONAL / SHARED. 소프트 딜리트 (deletedAt). PERSONAL은 ownerId 일치 검증 필수
Notice title, content, authorId 공지사항. POST 후 sendPushToAll().catch(console.error) 호출
Notification message, isRead, userId 앱 내 알림. 일정 초대, 연차 승인/반려 시 생성
PushSubscription endpoint, p256dh, auth, userId 웹 푸시 구독 정보. 410/404 응답 시 자동 삭제

⏰ Vercel Cron

vercel.json에 정의된 두 개의 Cron Job이 있습니다.

경로 스케줄 동작
/api/cron/daily-schedule-reminder 0 0 * * * (매일 00:00 UTC) 당일 일정이 있는 직원에게 알림 발송
/api/cron/reset-annual-leave 0 15 31 12 * (12/31 15:00 UTC = 1/1 00:00 KST) 전 직원 annualLeave를 15일로 초기화
TSCron 인증 — Authorization 헤더 검증 
export async function GET(req: Request) {
  const authHeader = req.headers.get("authorization")
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return Response.json({ error: "Unauthorized" }, { status: 401 })
  }
  // ... cron logic
}
⚠️

CRON_SECRET: Vercel 환경변수에 설정해야 합니다. Cron Job 요청 시 Vercel이 자동으로 Authorization: Bearer {CRON_SECRET} 헤더를 추가합니다.

💻 개발 명령어

Shell개발 명령어
# 개발 서버 (webpack 빌더 사용)
npm run dev          # next dev --webpack

# DB 관리
npm run db:migrate   # prisma migrate dev
npm run db:seed      # 초기 관리자 계정 생성
npm run db:studio    # Prisma Studio 실행

# 빌드 (Prisma 생성 포함)
npm run build        # prisma generate && next build --webpack
명령어 설명
npm run dev next dev --webpack — webpack 빌더로 개발 서버 실행
npm run db:migrate prisma migrate dev — DB 스키마 마이그레이션
npm run db:seed 초기 관리자 계정 생성 — admin@togate.com / admin1234!
npm run db:studio Prisma Studio (DB GUI) 실행
npm run build prisma generate && next build --webpack — Prisma 클라이언트 생성 후 빌드
💡

환경변수: .env.local에 DATABASE_URL, AUTH_SECRET, GEMINI_API_KEY, VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, CRON_SECRET를 설정하세요. 로그인 페이지 문서에서 전체 환경변수 목록을 확인할 수 있습니다.
