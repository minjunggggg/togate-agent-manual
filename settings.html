비밀번호 변경

초기 로그인 시 강제 실행되거나, 사용자가 직접 비밀번호를 변경하는 페이지입니다. proxy.ts가 passwordChanged 필드로 접근을 제어합니다.

마지막 업데이트 2026-05-14

📋 개요

/settings/password는 한 가지 진입 경로를 가집니다.

  • → 강제 진입: 관리자가 생성한 계정 최초 로그인 시 proxy.ts가 리다이렉트
💡

변경 성공 시 User.passwordChanged = true로 업데이트됩니다. 이후 proxy.ts는 해당 사용자를 강제 이동시키지 않습니다.

📁 파일 구조

src/app/settings/
password/
page.tsx ← 비밀번호 변경 폼 (클라이언트 컴포넌트)

src/app/api/
user/password/route.ts ← PATCH: 비밀번호 변경 + passwordChanged 업데이트

src/proxy.ts ← passwordChanged=false 시 이 페이지로 강제 이동
src/lib/auth.ts ← JWT 세션 (passwordChanged 포함)

page.tsx

TSXapp/settings/password/page.tsx
"use client"

import { useState } from "react"
import { signOut } from "next-auth/react"
import BackHeader from "@/components/layout/BackHeader"
import BoxInput from "@/components/common/BoxInput"
import Button from "@/components/common/Button"

export default function PasswordChangePage() {
  const [password, setPassword] = useState("")
  const [confirm, setConfirm] = useState("")
  const [saving, setSaving] = useState(false)
  const [error, setError] = useState("")

  const mismatch = confirm.length > 0 && password !== confirm
  const canSubmit = password.length >= 6 && confirm.length > 0 && !mismatch

  async function handleSubmit() {
    if (!canSubmit) return
    setSaving(true)
    setError("")
    try {
      const res = await fetch("/api/user/password", {
        method:  "PATCH",
        headers: { "Content-Type": "application/json" },
        body:    JSON.stringify({ password }),
      })
      if (res.ok) {
        await signOut({ redirect: false })
        window.location.href = "/login"
        return
      }
      const text = await res.text()
      let msg = "오류가 발생했습니다."
      try { msg = JSON.parse(text).error ?? msg } catch {}
      setError(msg)
    } catch {
      setError("서버에 연결할 수 없습니다.")
    } finally {
      setSaving(false)
    }
  }

  return (
    <div className="h-svh bg-white flex flex-col">
      <BackHeader title="비밀번호 변경"
        onBack={async () => { await signOut({ redirect: false }); window.location.href = "/login" }} />
      <div className="flex-1 flex flex-col px-5 pb-5">
        <p className="text-[18px] font-semibold text-[#454545] mt-[36px] mb-[26px]">
          초기 비밀번호 설정
        </p>
        <div className="flex flex-col gap-[24px] flex-1">
          <BoxInput label="비밀번호 입력" required type="password"
            value={password} onChange={setPassword}
            placeholder="비밀번호를 입력해 주세요" />
          <BoxInput label="비밀번호 확인" required type="password"
            value={confirm} onChange={setConfirm}
            placeholder="비밀번호를 다시 입력해 주세요"
            error={mismatch ? "비밀번호가 일치하지 않습니다." : error || undefined} />
        </div>
        <Button onClick={handleSubmit} disabled={!canSubmit} loading={saving}
          className="h-[52px] mt-6">
          {saving ? "변경 중..." : "비밀번호 변경"}
        </Button>
      </div>
    </div>
  )
}

🌐 API 라우트

Method 경로 설명
PATCH /api/user/password 새 비밀번호로 교체 및 passwordChanged=true 설정 (초기 설정이므로 현재 비밀번호 검증 없음)
TSapp/api/user/password/route.ts
import { auth } from "@/lib/auth"
import { prisma } from "@/lib/db"
import { NextResponse } from "next/server"
import bcrypt from "bcryptjs"

export async function PATCH(req: Request) {
  const session = await auth()
  if (!session?.user?.id)
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const { password } = await req.json()
  if (!password || password.length < 6)
    return NextResponse.json({ error: "비밀번호는 6자 이상이어야 합니다." }, { status: 400 })

  const hashed = await bcrypt.hash(password, 10)
  await prisma.user.update({
    where: { id: session.user.id },
    data: { password: hashed, passwordChanged: true },
  })

  return new NextResponse(null, { status: 204 })
}

✅ 유효성 검사

필드 규칙 에러 메시지
password 필수, 6자 이상 (클라이언트 + 서버 이중 검증) 비밀번호는 6자 이상이어야 합니다.
confirm password와 동일 비밀번호가 일치하지 않습니다.

🔒 보안

⚠️

JWT 세션 갱신: passwordChanged를 DB에서 업데이트한 뒤, JWT에 포함된 값도 반영되려면 세션 재발급이 필요합니다. Auth.js v5의 update()를 호출하거나, 재로그인을 유도하세요.

  • ✓ 초기 설정이므로 현재 비밀번호 검증 없이 바로 교체 (관리자 생성 계정 전용)
  • ✓ 새 비밀번호 bcryptjs (cost factor 10) 해싱 후 저장
  • ✓ passwordChanged=true 업데이트 후 proxy.ts 강제 이동 해제
  • ✓ 변경 성공 및 뒤로가기 시 signOut 후 /login 리다이렉트 (세션 즉시 무효화)
  • ✓ /settings/password는 로그인된 모든 사용자 접근 가능 (proxy.ts matcher 예외 처리)
