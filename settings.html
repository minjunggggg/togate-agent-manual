비밀번호 변경

초기 로그인 시 강제 실행되거나, 사용자가 직접 비밀번호를 변경하는 페이지입니다. proxy.ts가 passwordChanged 필드로 접근을 제어합니다.

마지막 업데이트 2026-05-14

📋 개요

/settings/password는 한 가지 진입 경로를 가집니다.

  • → 강제 진입: 관리자가 생성한 계정 최초 로그인 시 proxy.ts가 리다이렉트
💡

변경 성공 시 User.passwordChanged = true로 업데이트됩니다. 이후 proxy.ts는 해당 사용자를 강제 이동시키지 않습니다.

page.tsx

TSXapp/settings/password/page.tsx
"use client"

import { useState } from "react"
import { useRouter } from "next/navigation"
import { Input } from "@/components/common/Input"
import { Button } from "@/components/common/Button"
import { BottomButtonBar } from "@/components/common/BottomButtonBar"

export default function PasswordPage() {
  const router = useRouter()
  const [error, setError] = useState<string | null>(null)

  async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault()
    const fd = new FormData(e.currentTarget)

    if (fd.get("newPassword") !== fd.get("confirm")) {
      setError("새 비밀번호가 일치하지 않습니다.")
      return
    }

    const res = await fetch("/api/settings/password", {
      method:  "PATCH",
      headers: { "Content-Type": "application/json" },
      body:    JSON.stringify({
        currentPassword: fd.get("currentPassword"),
        newPassword:     fd.get("newPassword"),
      }),
    })

    if (!res.ok) {
      const { error } = await res.json()
      setError(error)
      return
    }
    router.push("/")
  }

  return (
    <form onSubmit={handleSubmit}>
      {error && <p role="alert">{error}</p>}
      <Input name="currentPassword" type="password" label="현재 비밀번호" />
      <Input name="newPassword"     type="password" label="새 비밀번호"    />
      <Input name="confirm"         type="password" label="새 비밀번호 확인" />
      <BottomButtonBar>
        <Button type="submit" variant="primary">변경하기</Button>
      </BottomButtonBar>
    </form>
  )
}

🌐 API 라우트

Method 경로 설명
PATCH /api/settings/password 현재 비밀번호 검증 후 새 비밀번호로 교체, passwordChanged=true 설정
TSapp/api/settings/password/route.ts
import { NextResponse } from "next/server"
import { auth } from "@/lib/auth"
import { db } from "@/lib/db"
import { hashPassword, verifyPassword } from "@/lib/crypto"

export async function PATCH(req: Request) {
  const session = await auth()
  if (!session?.user)
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const { currentPassword, newPassword } = await req.json()

  const user = await db.user.findUnique({ where: { id: session.user.id } })
  if (!user) return NextResponse.json({ error: "Not Found" }, { status: 404 })

  const isValid = await verifyPassword(currentPassword, user.hashedPassword)
  if (!isValid)
    return NextResponse.json({ error: "현재 비밀번호가 올바르지 않습니다." }, { status: 400 })

  await db.user.update({
    where: { id: session.user.id },
    data: {
      hashedPassword:  await hashPassword(newPassword),
      passwordChanged: true,
    },
  })

  return NextResponse.json({ ok: true })
}

✅ 유효성 검사

필드 규칙 에러 메시지
currentPassword 필수, DB 해시와 일치 현재 비밀번호가 올바르지 않습니다.
newPassword 필수, 8자 이상 비밀번호는 8자 이상이어야 합니다.
confirm newPassword와 동일 새 비밀번호가 일치하지 않습니다.

⚠️

JWT 세션 갱신: passwordChanged를 DB에서 업데이트한 뒤, JWT에 포함된 값도 반영되려면 세션 재발급이 필요합니다. Auth.js v5의 update()를 호출하거나, 재로그인을 유도하세요.

  • ✓ currentPassword 서버에서 bcrypt 검증 (클라이언트 생략 가능)
  • ✓ 새 비밀번호 bcrypt (cost factor 12) 해싱 후 저장
  • ✓ passwordChanged=true 업데이트 후 proxy.ts 강제 이동 해제
  • ✓ /settings/password는 로그인된 모든 사용자 접근 가능 (proxy.ts matcher 예외 처리)
