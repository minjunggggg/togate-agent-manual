사용자 등록

Togate는 사내앱으로, 관리자(ADMIN)가 직원 계정을 생성합니다. 공개 자가 회원가입은 없습니다.

마지막 업데이트 2026-05-14

📋 개요

사용자 등록은 /users/new와 /users/new/profile에서 이뤄집니다. ADMIN 권한을 가진 계정만 접근할 수 있으며, 등록된 직원은 초기 비밀번호로 첫 로그인 후 반드시 비밀번호를 변경해야 합니다.

💡

설계 원칙: 공개 가입 없음. 관리자가 User + Employee를 생성하고, 직원은 첫 로그인 시 /settings/password로 강제 이동되어 비밀번호를 교체합니다. src/proxy.ts가 이 흐름을 제어합니다.

주요 기능

  • → ADMIN만 /users/new 접근 가능 (proxy.ts 권한 체크)
  • → User (이메일·비밀번호·role) + Employee (이름·전화·직책) 동시 생성
  • → User.passwordChanged = false로 초기 설정
  • → 직원 첫 로그인 시 /settings/password 강제 이동
  • → 연차 잔여일수(annualLeave) 초기값 설정

🧩 컴포넌트

/users/new/page.tsx — 계정 생성 1단계

TSXapp/users/new/page.tsx
"use client"

import { useState } from "react"
import { useRouter } from "next/navigation"
import { Button } from "@/components/common/Button"
import { Input } from "@/components/common/Input"

export default function NewUserPage() {
  const router = useRouter()
  const [isLoading, setIsLoading] = useState(false)

  async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault()
    setIsLoading(true)

    const formData = new FormData(e.currentTarget)
    const res = await fetch("/api/users", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        email:    formData.get("email"),
        password: formData.get("password"),
        role:     formData.get("role"),
      }),
    })

    if (res.ok) {
      const { id } = await res.json()
      router.push(`/users/new/profile?userId=${id}`)
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      {/* email · password · role 입력 */}
    </form>
  )
}

POST /api/users

TSapp/api/users/route.ts
import { NextResponse } from "next/server"
import { auth } from "@/lib/auth"
import { db } from "@/lib/db"
import { hashPassword } from "@/lib/crypto"

export async function POST(req: Request) {
  const session = await auth()
  if (session?.user.role !== "ADMIN")
    return NextResponse.json({ error: "Forbidden" }, { status: 403 })

  const { email, password, role } = await req.json()

  const hashedPassword = await hashPassword(password)
  const user = await db.user.create({
    data: {
      email,
      hashedPassword,
      role,
      passwordChanged: false,  // 첫 로그인 시 강제 변경 트리거
      annualLeave: 15,
    },
  })

  return NextResponse.json({ id: user.id }, { status: 201 })
}

🔐 최초 로그인 — 비밀번호 강제 변경

관리자가 생성한 계정으로 직원이 처음 로그인하면 src/proxy.ts가 passwordChanged 필드를 확인해 /settings/password로 강제 이동합니다.

TSsrc/proxy.ts — passwordChanged 체크
import { NextRequest, NextResponse } from "next/server"
import { auth } from "@/lib/auth"

export async function proxy(req: NextRequest) {
  const session = await auth()
  const { pathname } = req.nextUrl

  if (!session?.user)
    return NextResponse.redirect(new URL("/login", req.url))

  if (!session.user.passwordChanged && pathname !== "/settings/password")
    return NextResponse.redirect(new URL("/settings/password", req.url))

  if (pathname.startsWith("/admin") && session.user.role !== "ADMIN")
    return NextResponse.redirect(new URL("/", req.url))
}

export const config = {
  matcher: ["/((?!login|api/auth|_next|.*\\..*).*)"],
}
💡

Next.js 16 proxy 규칙: 파일 이름은 반드시 src/proxy.ts이고, export 이름은 proxy입니다. middleware로 export하면 동작하지 않습니다.

🗄️ DB 모델

사용자 등록 시 User와 Employee 모델이 1:1로 함께 생성됩니다.

TSprisma/schema.prisma — User · Employee
model User {
  id               String    @id @default(cuid())
  email            String    @unique
  hashedPassword   String
  role             Role      @default(EMPLOYEE)
  annualLeave      Int       @default(15)
  passwordChanged  Boolean   @default(false)
  employee         Employee?
}

model Employee {
  id       String  @id @default(cuid())
  name     String
  phone    String?
  position String           // 직책
  userId   String  @unique
  user     User    @relation(fields: [userId], references: [id])
}

enum Role { ADMIN  EMPLOYEE }

Prisma 7 필수 사항

  • ✓ import 경로: @/generated/prisma/client
  • ✓ @prisma/adapter-pg 어댑터 필수 — 없으면 런타임 에러
  • ✓ prisma.config.ts에서 DATABASE_URL 처리 (datasource에 직접 url 불필요)
  • ✓ 스키마 변경 후 반드시 npm run db:migrate 실행

🔒 보안

🚨

API 레벨 권한 체크 필수: proxy.ts는 페이지 라우트를 보호하지만, API Route 핸들러도 반드시 auth()로 세션을 확인하고 role을 검증해야 합니다.

보안 체크리스트

  • ✓ POST /api/users — ADMIN role 서버 검증 필수
  • ✓ 비밀번호 bcrypt (cost factor 12) 해싱
  • ✓ proxy.ts — ADMIN 전용 라우트(/users/new, /admin/*) 접근 제어
  • ✓ passwordChanged=false 계정 → /settings/password 강제 이동
  • ✓ 이메일 중복 시 409, 구체적 계정 존재 여부는 노출 금지
  • ✓ 초기 임시 비밀번호는 관리자가 직접 직원에게 전달 (시스템 발송 없음)
