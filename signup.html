사용자 등록

Togate는 사내앱으로, 관리자(ADMIN)가 직원 계정을 생성합니다. 공개 자가 회원가입은 없습니다.

마지막 업데이트 2026-05-19

📋 개요

src/app/(app)/users/page.tsx는 사용자 목록 조회·삭제를 담당하는 서버 컴포넌트입니다. 데이터 패치는 서버에서 수행하고, 검색·삭제 인터랙션은 UsersClient 클라이언트 컴포넌트에 위임합니다. 신규 계정 생성은 /users/new → /users/new/profile 2단계로 진행됩니다.

💡

설계 원칙: 공개 가입 없음. 관리자가 User + Employee를 생성하고, 직원은 첫 로그인 시 /settings/password로 강제 이동되어 비밀번호를 교체합니다. src/proxy.ts가 이 흐름을 제어합니다.

주요 기능

  • → 서버에서 전체 사용자 목록 조회 후 UsersClient에 전달
  • → 클라이언트 이름 검색 (client-side filter)
  • → 계정 삭제 — Dialog 확인 → DELETE /api/admin/users/[id] → 완료 Dialog
  • → 신규 계정 생성: BackHeader 우측 + 버튼 → /users/new
  • → User.passwordChanged = false로 초기 설정 → 첫 로그인 시 /settings/password 강제 이동
  • → ADMIN 배지(AdminBadge) 표시

UsersClient.tsx — 클라이언트 컴포넌트

TSXapp/(app)/users/UsersClient.tsx
"use client"

import { useState } from "react"
import Dialog from "@/components/common/Dialog"
import EmptyState from "@/components/common/EmptyState"
import AdminBadge from "@/components/common/AdminBadge"

interface Employee {
  id: string; name: string; phone: string | null;
  position: string | null; job: string | null;
}
interface User {
  id: string; name: string; email: string; role: string; employee: Employee | null;
}

export default function UsersClient({ users }: { users: User[] }) {
  const [query, setQuery] = useState("")
  const [list, setList] = useState(users)
  const [deleteTarget, setDeleteTarget] = useState<{ id: string; name: string } | null>(null)
  const [deletedName, setDeletedName] = useState<string | null>(null)

  async function handleDelete() {
    if (!deleteTarget) return
    await fetch(`/api/admin/users/${deleteTarget.id}`, { method: "DELETE" })
    setList((prev) => prev.filter((u) => u.id !== deleteTarget.id))
    setDeletedName(deleteTarget.name)
    setDeleteTarget(null)
  }

  const filtered = query.trim()
    ? list.filter((u) => u.name.includes(query.trim()))
    : list

  return (
    <>
      {/* 삭제 확인 Dialog */}
      {deleteTarget && (
        <Dialog
          message={`${deleteTarget.name}님의 계정을 삭제하시겠습니까?`}
          buttons={[
            { label: "취소", onClick: () => setDeleteTarget(null) },
            { label: "삭제", variant: "primary", onClick: handleDelete },
          ]}
        />
      )}
      {/* 삭제 완료 Dialog */}
      {deletedName && (
        <Dialog
          message={`${deletedName}님의 계정이 삭제되었습니다.`}
          buttons={[{ label: "확인", onClick: () => setDeletedName(null) }]}
        />
      )}
      {/* 이름 검색 · 목록 렌더링 */}
      {/* filtered.length === 0 → EmptyState */}
      {/* 각 카드: position / job · 이름 · 이메일 · 삭제 버튼 */}
    </>
  )
}
🚨

confirm() 금지: 삭제·확인 다이얼로그는 반드시 Dialog 컴포넌트를 사용하세요. 브라우저 기본 confirm()은 사용 금지입니다. UsersClient는 삭제 확인과 삭제 완료 두 개의 Dialog를 별도 state로 관리합니다.

🌐 API 라우트

💡

목록 조회는 API 없음: users/page.tsx에서 Prisma로 직접 조회합니다. API Route는 생성(POST)만 존재하며, 수정·삭제는 /api/contacts/[id]에서 처리합니다.

Method경로설명권한
POST/api/admin/users신규 User + Employee 생성ADMIN
TSapp/api/admin/users/route.ts
export async function POST(req: Request) {
  const session = await auth()
  if (session?.user?.role !== "ADMIN")
    return NextResponse.json({ error: "Forbidden" }, { status: 403 })

  const { name, email, password, role, annualLeave,
          position, job, phone, residence } = await req.json()

  if (!name || !email || !password)
    return NextResponse.json({ error: "필수 항목을 입력해주세요." }, { status: 400 })

  const exists = await prisma.user.findUnique({ where: { email } })
  if (exists)
    return NextResponse.json({ error: "이미 사용 중인 이메일입니다." }, { status: 400 })

  const hashedPassword = await bcrypt.hash(password, 12)
  const user = await prisma.user.create({
    data: {
      name, email, password: hashedPassword,
      role: role ?? "EMPLOYEE",
      annualLeave: annualLeave ?? 15,
      employee: {
        create: { name, email, position: position ?? null,
                   job: job ?? null, phone: phone ?? null, residence: residence ?? null },
      },
    },
  })

  // 기존 사용자 전원에게 새 직원 알림 + 푸시 발송
  const existingUsers = await prisma.user.findMany({
    where: { id: { not: user.id } }, select: { id: true },
  })
  await prisma.notification.createMany({ data: existingUsers.map((u) => ({
    userId: u.id, type: "EMPLOYEE",
    title: pushMessages.contact.title,
    message: pushMessages.contact.created(), referenceId: user.id,
  })) })
  await Promise.allSettled(existingUsers.map((u) =>
    sendPushToUser(u.id, { title: pushMessages.contact.title,
      body: pushMessages.contact.created(), url: "/contacts" })
  ))

  return NextResponse.json(
    { id: user.id, name: user.name, email: user.email },
    { status: 201 }
  )
}

🔐 최초 로그인 — 비밀번호 강제 변경

관리자가 생성한 계정으로 직원이 처음 로그인하면 src/proxy.ts가 passwordChanged 필드를 확인해 /settings/password로 강제 이동합니다.

TSsrc/proxy.ts — passwordChanged 체크
import { NextRequest, NextResponse } from "next/server"
import { auth } from "@/lib/auth"

export async function proxy(req: NextRequest) {
  const session = await auth()
  const { pathname } = req.nextUrl

  if (!session?.user)
    return NextResponse.redirect(new URL("/login", req.url))

  if (!session.user.passwordChanged && pathname !== "/settings/password")
    return NextResponse.redirect(new URL("/settings/password", req.url))

  if (pathname.startsWith("/admin") && session.user.role !== "ADMIN")
    return NextResponse.redirect(new URL("/", req.url))
}

export const config = {
  matcher: ["/((?!login|api/auth|_next|.*\\..*).*)"],
}
💡

Next.js 16 proxy 규칙: 파일 이름은 반드시 src/proxy.ts이고, export 이름은 proxy입니다. middleware로 export하면 동작하지 않습니다.

🗄️ DB 모델

사용자 등록 시 User와 Employee 모델이 1:1로 함께 생성됩니다.

TSprisma/schema.prisma — User · Employee
model User {
  id               String    @id @default(cuid())
  name             String
  email            String    @unique
  password         String              // bcrypt 해시
  role             Role      @default(EMPLOYEE)
  annualLeave      Int       @default(15)
  passwordChanged  Boolean   @default(false)
  employee         Employee?
}

model Employee {
  id        String    @id @default(cuid())
  name      String
  email     String?
  phone     String?
  position  String?
  job       String?
  residence String?
  userId    String?  @unique
  user      User?   @relation(fields: [userId], references: [id], onDelete: Cascade)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

enum Role { ADMIN  EMPLOYEE }

Prisma 7 필수 사항

  • ✓ import 경로: @/generated/prisma/client

  • ✓ prisma.config.ts에서 DATABASE_URL 처리 (datasource에 직접 url 불필요)
  • ✓ 스키마 변경 후 반드시 npm run db:migrate 실행
