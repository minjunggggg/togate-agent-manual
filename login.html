로그인 페이지

Togate 사내앱의 인증 진입점입니다. 이메일/비밀번호 로그인과 초기 비밀번호 강제 변경 플로우를 다룹니다.

마지막 업데이트 2026-05-15

📋 개요

로그인 페이지는 src/app/login/page.tsx 에 위치하며, Next.js App Router 기반의 서버 컴포넌트로 렌더링됩니다. 폼 인터랙션은 Server Action(actions.ts)으로 처리합니다.

💡

설계 원칙: 페이지 컴포넌트는 서버 컴포넌트로 유지해 메타데이터·레이아웃을 서버에서 처리하고, 폼 인터랙션만 클라이언트 컴포넌트로 분리합니다.

주요 기능

  • → 이메일 + 비밀번호 로그인 (Credentials Provider)
  • → JWT 세션 기반 인증 (Auth.js v5)
  • → 초기 비밀번호 미변경 시 /settings/password 강제 이동
  • → 로그인 후 callbackUrl 리다이렉트
  • → 클라이언트·서버 양측 입력값 유효성 검사

📁 파일 구조

로그인 관련 파일은 src/app/login/에 위치하며, 인증 설정은 src/lib/에서 관리합니다.

🧩 컴포넌트

page.tsx — 서버 컴포넌트

TSXsrc/app/login/page.tsx
import { Metadata } from "next"
import { redirect } from "next/navigation"
import { auth } from "@/lib/auth"

export const metadata: Metadata = {
  title: "로그인 | Togate",
  description: "Togate 계정으로 로그인하세요",
}

export default async function LoginPage() {
  const session = await auth()
  if (session?.user) redirect("/")

  return (
    <main className="flex min-h-screen items-center justify-center">
      {/* 폼 렌더링 — actions.ts의 Server Action을 form action으로 연결 */}
    </main>
  )
}

actions.ts — Server Action

TSsrc/app/login/actions.ts
"use server"

import { signIn } from "@/lib/auth"
import { redirect } from "next/navigation"

export async function login(formData: FormData) {
  const email    = formData.get("email")    as string
  const password = formData.get("password") as string

  await signIn("credentials", { email, password, redirect: false })
  redirect("/")
}

NextAuth가 자동으로 아래 엔드포인트를 생성합니다.

Method경로설명

lib/auth.ts — NextAuth 설정

TSsrc/lib/auth.ts
import NextAuth from "next-auth"
import Credentials from "next-auth/providers/credentials"
import { prisma } from "@/lib/db"

export const { handlers, auth, signIn, signOut } = NextAuth({
  session: { strategy: "jwt" },
  providers: [
    Credentials({
      async authorize(credentials) {
        const user = await prisma.user.findUnique({
          where: { email: credentials.email as string },
        })
        if (!user) return null
        // bcrypt 검증 후 user 반환
        return user
      },
    }),
  ],
  pages: {
    signIn: "/login",
    error:  "/login",
  },
})
💡

auth.config.ts 분리: Credentials Provider 설정, callbacks, JWT 로직은 src/lib/auth.config.ts로 분리해 관리합니다. auth.ts는 NextAuth(authConfig)를 호출해 handlers를 export하는 역할만 합니다.

✅ 유효성 검사

Server Action(actions.ts) 내부에서 Zod로 입력값을 검증합니다.

TSsrc/app/login/actions.ts — Zod 검증
"use server"

import { z } from "zod"
import { signIn } from "@/lib/auth"

const schema = z.object({
  email:    z.string().min(1, "이메일을 입력해주세요.").email("올바른 이메일 형식이 아닙니다."),
  password: z.string().min(8, "비밀번호는 8자 이상이어야 합니다."),
})

export async function login(formData: FormData) {
  const parsed = schema.safeParse({
    email:    formData.get("email"),
    password: formData.get("password"),
  })
  if (!parsed.success) return { error: parsed.error.issues[0].message }
  await signIn("credentials", parsed.data)
}
필드규칙에러 메시지
email필수, 이메일 형식이메일을 입력해주세요 / 올바른 이메일 형식이 아닙니다
password필수, 최소 8자비밀번호를 입력해주세요 / 비밀번호는 8자 이상이어야 합니다

🔒 보안

🚨

절대 하지 말 것: 인증 실패 시 "이메일이 존재하지 않습니다" / "비밀번호가 틀렸습니다"를 구분해 노출하지 마세요. 계정 열거(Account Enumeration) 공격에 악용됩니다.

  • ✓ 비밀번호는 bcrypt (cost factor 12) 로 해싱
  • ✓ HTTPS 환경에서만 쿠키 전송 (Secure 플래그)
  • ✓ CSRF 토큰 — NextAuth 내장 보호 사용
  • ✓ proxy.ts에서 인증 게이트 + 초기 비밀번호 강제 변경 처리
  • ✓ 에러 메시지 통일 — 계정 열거 방지
  • ✓ AUTH_SECRET 32자 이상 랜덤 문자열 사용

⚙️ 환경변수

인증에 관련된 주요 환경변수는 아래와 같습니다. 전체 목록은 프로젝트 개요 → 환경변수에서 확인하세요.

변수설명
AUTH_SECRET JWT 서명·암호화 키. 32자 이상 랜덤 문자열 필수
AUTH_URL 로컬 개발 시 기준 URL. Vercel 배포 시 자동 주입 — 직접 설정 시 Preview URL 충돌 주의
AUTH_TRUST_HOST Vercel·역방향 프록시 환경에서 true로 설정
DATABASE_URL PostgreSQL 연결 문자열. Prisma 7 + @prisma/adapter-pg가 사용
💡

Vercel 배포 팁: AUTH_URL은 설정하지 않고 AUTH_TRUST_HOST=true만 설정하는 것이 권장됩니다.
