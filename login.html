로그인 페이지

Togate 사내앱의 인증 진입점입니다. 이메일/비밀번호 로그인과 초기 비밀번호 강제 변경 플로우를 다룹니다.

마지막 업데이트 2026-05-14

📋 개요

로그인 페이지는 /app/(auth)/login/page.tsx 에 위치하며, Next.js App Router 기반의 서버 컴포넌트로 렌더링됩니다. 클라이언트 상태 관리는 LoginForm 클라이언트 컴포넌트가 담당합니다.

💡

설계 원칙: 페이지 컴포넌트는 서버 컴포넌트로 유지해 메타데이터·레이아웃을 서버에서 처리하고, 폼 인터랙션만 클라이언트 컴포넌트로 분리합니다.

주요 기능

  • → 이메일 + 비밀번호 로그인 (Credentials Provider)
  • → JWT 세션 기반 인증 (Auth.js v5)
  • → 초기 비밀번호 미변경 시 /settings/password 강제 이동
  • → 로그인 후 callbackUrl 리다이렉트
  • → 클라이언트·서버 양측 입력값 유효성 검사

📁 파일 구조

인증 관련 파일은 (auth) 라우트 그룹으로 묶어 공통 레이아웃을 공유합니다.

🧩 컴포넌트

page.tsx — 서버 컴포넌트

TSXapp/(auth)/login/page.tsx 
import { Metadata } from "next"
import { redirect } from "next/navigation"
import { auth } from "@/lib/auth"
import { LoginForm } from "@/components/auth/LoginForm"

export const metadata: Metadata = {
  title: "로그인 | Togate",
  description: "Togate 계정으로 로그인하세요",
}

export default async function LoginPage() {
  const session = await auth()

  // 이미 로그인된 사용자는 홈으로 리다이렉트
  if (session?.user) redirect("/")

  return (
    <main className="flex min-h-screen items-center justify-center">
      <LoginForm />
    </main>
  )
}

LoginForm.tsx — 클라이언트 컴포넌트

TSXcomponents/auth/LoginForm.tsx
"use client"

import { useState } from "react"
import { signIn } from "next-auth/react"
import { useRouter, useSearchParams } from "next/navigation"
import { loginSchema } from "@/lib/validations/auth"

export function LoginForm() {
  const router = useRouter()
  const searchParams = useSearchParams()
  const callbackUrl = searchParams.get("callbackUrl") ?? "/"

  const [isLoading, setIsLoading] = useState(false)
  const [error, setError] = useState<string | null>(null)

  async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault()
    const formData = new FormData(e.currentTarget)
    const raw = {
      email:    formData.get("email"),
      password: formData.get("password"),
    }

    const parsed = loginSchema.safeParse(raw)
    if (!parsed.success) {
      setError(parsed.error.issues[0].message)
      return
    }

    setIsLoading(true)
    setError(null)

    const result = await signIn("credentials", {
      ...parsed.data,
      redirect: false,
    })

    if (result?.error) {
      setError("이메일 또는 비밀번호가 올바르지 않습니다.")
      setIsLoading(false)
      return
    }

    router.push(callbackUrl)
  }

  return (
    <form onSubmit={handleSubmit} noValidate>
      {error && <p role="alert">{error}</p>}
      {/* 입력 필드는 하위 컴포넌트로 분리 */}
    </form>
  )
}

lib/auth.ts — NextAuth 설정

TSsrc/lib/auth.ts
import NextAuth from "next-auth"
import Credentials from "next-auth/providers/credentials"
import { loginSchema } from "@/lib/validations/auth"
import { verifyPassword } from "@/lib/crypto"
import { db } from "@/lib/db"

export const { handlers, auth, signIn, signOut } = NextAuth({
  session: { strategy: "jwt" },
  providers: [
    Credentials({
      async authorize(credentials) {
        const parsed = loginSchema.safeParse(credentials)
        if (!parsed.success) return null

        const user = await db.user.findUnique({
          where: { email: parsed.data.email },
        })
        if (!user) return null

        const isValid = await verifyPassword(
          parsed.data.password, user.hashedPassword
        )
        return isValid ? user : null
      },
    }),
  ],
  pages: {
    signIn: "/login",
    error:  "/login",
  },
})

✅ 유효성 검사

클라이언트와 서버 모두 동일한 Zod 스키마를 사용합니다.

TSsrc/lib/validations/auth.ts
import { z } from "zod"

export const loginSchema = z.object({
  email:    z.string().min(1, "이메일을 입력해주세요.").email("올바른 이메일 형식이 아닙니다."),
  password: z.string().min(1, "비밀번호를 입력해주세요.").min(8, "비밀번호는 8자 이상이어야 합니다."),
})

export type LoginInput = z.infer<typeof loginSchema>
필드규칙에러 메시지
email필수, 이메일 형식이메일을 입력해주세요 / 올바른 이메일 형식이 아닙니다
password필수, 최소 8자비밀번호를 입력해주세요 / 비밀번호는 8자 이상이어야 합니다

🔒 보안

🚨

절대 하지 말 것: 인증 실패 시 "이메일이 존재하지 않습니다" / "비밀번호가 틀렸습니다"를 구분해 노출하지 마세요. 계정 열거(Account Enumeration) 공격에 악용됩니다.

  • ✓ 비밀번호는 bcrypt (cost factor 12) 로 해싱
  • ✓ HTTPS 환경에서만 쿠키 전송 (Secure 플래그)
  • ✓ CSRF 토큰 — NextAuth 내장 보호 사용
  • ✓ proxy.ts에서 인증 게이트 + 초기 비밀번호 강제 변경 처리
  • ✓ 에러 메시지 통일 — 계정 열거 방지
  • ✓ AUTH_SECRET 32자 이상 랜덤 문자열 사용

⚙️ 환경변수

ENV.env.local
# NextAuth (Auth.js v5)
AUTH_SECRET=your-32-char-random-secret
NEXTAUTH_URL=http://localhost:3000

# Database (Prisma 7 + PostgreSQL)
DATABASE_URL=postgresql://user:pass@localhost:5432/togate
⚠️

Vercel 배포 시: NEXTAUTH_URL은 Vercel이 자동으로 주입합니다. 직접 설정하면 Preview 환경에서 URL 충돌이 발생할 수 있습니다.
