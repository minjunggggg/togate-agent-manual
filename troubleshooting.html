트러블슈팅

개발·운영 중 자주 발생하는 오류와 해결 방법을 모아 정리합니다. 오류를 마주치면 여기서 먼저 찾아보세요.

마지막 업데이트 2026-05-14

🔧 환경 세팅 문제

❌ npm install 후 실행 시 모듈 오류

🚨

증상: Cannot find module 'XXX' 또는 Module not found

BASH해결: node_modules 초기화
# node_modules와 lock 파일 삭제 후 재설치
rm -rf node_modules .next
npm ci             # package-lock.json 기준으로 정확히 설치
npm run dev

❌ Next.js 개발 서버가 실행은 되는데 화면이 빈 페이지

⚠️

원인: .env.local 파일이 없거나 AUTH_SECRET이 비어 있을 때 발생합니다.

BASH해결: 환경변수 확인
# .env.local 존재 여부 확인
ls -la .env.local

# AUTH_SECRET 값이 있는지 확인 (값 자체는 노출 주의)
grep AUTH_SECRET .env.local

❌ NEXT_PUBLIC_VAPID_PUBLIC_KEY is not defined

BASH해결: VAPID 키 생성 및 env 추가
# VAPID 키 쌍 생성
npx web-push generate-vapid-keys

# 출력된 값을 .env.local에 추가
NEXT_PUBLIC_VAPID_PUBLIC_KEY=xxxxxxx
VAPID_PRIVATE_KEY=xxxxxxx
VAPID_MAILTO=mailto:admin@togate.com

🗄️ Prisma / DB 문제

❌ @prisma/adapter-pg is required 런타임 에러

🚨

원인: Prisma 7은 어댑터 주입이 필수입니다. src/lib/db.ts에서 PrismaPg 어댑터 없이 PrismaClient를 직접 생성하면 발생합니다.

TS올바른 db.ts 패턴
import { PrismaClient } from "@/generated/prisma/client"  // ← @prisma/client 아님!
import { PrismaPg } from "@prisma/adapter-pg"

function createPrismaClient() {
  const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL! })
  return new PrismaClient({ adapter })   // ← adapter 필수
}

❌ Cannot find module '@/generated/prisma/client'

BASH해결: Prisma 클라이언트 재생성
# Prisma 클라이언트 수동 생성
npx prisma generate

# 또는 빌드 명령어 (prisma generate 포함됨)
npm run build
💡

Import 경로는 반드시 @/generated/prisma/client입니다. @prisma/client를 사용하면 Prisma 7에서 동작하지 않습니다. prisma/schema.prisma의 output 설정을 확인하세요.

❌ npm run db:migrate 실행 시 연결 오류

BASH해결: DATABASE_URL 확인
# DB URL 형식 확인
# postgresql://USER:PASSWORD@HOST:PORT/DBNAME
echo $DATABASE_URL

# 로컬 PostgreSQL 실행 여부 확인
pg_isready -h localhost -p 5432

❌ 소프트 딜리트된 폴더가 목록에 노출됨

TS원인: deletedAt 필터 누락
// ❌ 잘못된 쿼리 — 삭제된 폴더도 반환됨
const folders = await db.meetingFolder.findMany({
  where: { ownerId: userId }
})

// ✅ 올바른 쿼리 — deletedAt: null 필터 필수
const folders = await db.meetingFolder.findMany({
  where: { ownerId: userId, deletedAt: null }   // ← 반드시 추가
})

🔑 인증 문제

❌ 로그인 후 계속 로그인 페이지로 리다이렉트됨

⚠️

원인 1: AUTH_SECRET이 설정되지 않았거나 짧습니다. JWT 서명에 실패해 세션이 생성되지 않습니다.

⚠️

원인 2: Vercel 환경에서 AUTH_TRUST_HOST=true가 없으면 Host 헤더 불신으로 세션 쿠키가 설정되지 않습니다.

BASH해결: 환경변수 점검
# 1. AUTH_SECRET 길이 확인 (32자 이상)
echo -n "$AUTH_SECRET" | wc -c

# 2. .env.local에 아래 항목 확인
AUTH_SECRET=...32자 이상 랜덤 문자열...
AUTH_TRUST_HOST=true

❌ proxy.ts 수정 후 미들웨어가 동작 안 함

TS원인: export 이름 오류
// ❌ 잘못된 export — Next.js 16에서 인식 안 됨
export default middleware
export const config = { matcher: "..." }

// ✅ 올바른 export — proxy와 config를 named export
export { auth as proxy } from "@/lib/auth"
export const config = { matcher: "..." }

❌ 초기 비밀번호 변경 강제 이동이 안 됨

TS점검: proxy.ts의 passwordChanged 체크
// proxy.ts 내 아래 로직이 있는지 확인
if (session?.user && !session.user.passwordChanged &&
    !req.nextUrl.pathname.startsWith("/settings/password")) {
  return NextResponse.redirect(new URL("/settings/password", req.url))
}

🔔 웹 푸시 알림 문제

❌ 푸시 알림이 전혀 오지 않음

⚠️

체크리스트: 브라우저 알림 권한 허용 여부 → VAPID 키 일치 여부 → Service Worker 등록 여부 → DB의 PushSubscription 레코드 존재 여부를 순서대로 확인하세요.

BASH디버그: 브라우저 콘솔 확인
# 브라우저 개발자 도구 → Application → Service Workers
# → Status가 "activated"인지 확인

# 또는 콘솔에서 구독 상태 확인
const reg = await navigator.serviceWorker.ready
const sub = await reg.pushManager.getSubscription()
console.log(sub)  // null이면 구독되지 않은 상태

❌ 푸시 발송 시 410 Gone 에러 로그

💡

정상 동작입니다. 410은 브라우저가 구독을 취소했을 때 푸시 서버가 반환하는 코드입니다. src/lib/push.ts의 sendPushToUser가 자동으로 해당 구독을 DB에서 삭제합니다. 별도 조치 불필요합니다.

❌ iOS Safari에서 푸시가 오지 않음

⚠️

원인: iOS 16.4 미만 Safari는 웹 푸시를 지원하지 않습니다. iOS 16.4 이상이어도 사용자가 직접 홈 화면에 앱을 추가(Add to Home Screen)한 PWA에서만 푸시가 동작합니다. Safari 브라우저 탭에서는 지원되지 않습니다.

🤖 AI 회의록 문제

❌ 음성 전사 후 "처리 실패" 오류

BASH해결: API 키 및 오디오 형식 확인
# 1. Deepgram API 키 유효성 확인
curl -X GET https://api.deepgram.com/v1/projects \
  -H "Authorization: Token $DEEPGRAM_API_KEY"

# 2. Gemini API 키 유효성 확인
curl "https://generativelanguage.googleapis.com/v1/models?key=$GEMINI_API_KEY"
💡

지원 오디오 형식: Deepgram은 mp3, mp4, wav, webm 등 주요 형식을 지원합니다. 브라우저 MediaRecorder가 출력하는 형식(대부분 audio/webm)은 정상 지원됩니다.

❌ 화자 분리(speakerMap)가 비어 있거나 이상함

⚠️

원인: 음성 품질이 낮거나 발화자가 1명일 때 화자 분리 결과가 부정확할 수 있습니다. AI 모델의 한계이며 코드 문제가 아닙니다. speakerMap을 표시하는 UI에서 빈 값 예외 처리가 되어 있는지 확인하세요.

TSspeakerMap null 안전 처리
// speakerMap은 Json 타입 — 항상 타입 가드 필요
const speakerMap = minute.speakerMap as Record<string, string[]> ?? {}
const speakers = Object.keys(speakerMap)
if (speakers.length === 0) {
  // 화자 없음 처리
}

🚢 배포 문제

❌ Vercel 빌드 실패 — prisma generate 오류

BASH해결: postinstall 스크립트 확인
# package.json에 postinstall 추가 (Vercel은 npm install 후 자동 실행)
{
  "scripts": {
    "postinstall": "prisma generate",
    "build": "prisma generate && next build --webpack"
  }
}

원인: Vercel 환경변수에 AUTH_URL이 하드코딩되어 있으면 Preview 배포의 동적 URL과 충돌합니다.

BASH해결: Vercel에서 AUTH_URL 삭제
# Vercel 대시보드 → Settings → Environment Variables
# AUTH_URL 항목 삭제 (AUTH_TRUST_HOST=true 만으로 충분)

📱 iOS 빌드 문제

❌ npx cap sync ios 실행 후 Xcode 빌드 오류

BASH해결: 웹 빌드 먼저 실행
# cap sync 전에 반드시 웹 빌드가 선행되어야 합니다
npm run build       # 먼저 실행
npx cap sync ios   # 이후 실행
npx cap open ios

❌ 인증서 만료로 Archive 실패

⚠️

Xcode → Signing & Capabilities에서 팀이 올바르게 설정되어 있는지 확인하세요. 인증서가 만료된 경우 Apple Developer 계정에서 갱신 후 Xcode에서 재다운로드가 필요합니다. 자세한 내용은 외부 서비스 → Apple Developer를 참고하세요.

❌ 실기기에서 푸시가 오지 않음 (시뮬레이터는 정상)

💡

iOS 시뮬레이터는 웹 푸시를 완전히 지원하지 않습니다. 웹 푸시 관련 기능은 반드시 실기기(iPhone)에서 테스트하세요. 또한 iOS Safari PWA 형태(홈 화면에 추가)가 아닌 Capacitor 네이티브 앱에서 웹 푸시 동작 방식이 다를 수 있습니다.
