비상연락망

직원 연락처 디렉토리입니다. Employee 모델을 기반으로 이름·전화번호·직책을 조회하고 관리합니다.

마지막 업데이트 2026-05-14

📋 개요

  • → /contacts — 전체 직원 목록 (검색·필터)
  • → /contacts/[id] — 직원 상세 정보
  • → /contacts/[id]/edit — 직원 정보 수정 (ADMIN 전용)
💡

모든 로그인 사용자가 목록·상세를 조회하고 정보를 수정할 수 있습니다. 삭제는 ADMIN만 가능합니다.

🌐 API 라우트

Method경로설명권한
GET/api/contacts전체 직원 목록 (이름·직책·전화 포함)전체

TSapp/api/contacts/[id]/route.ts
export async function PATCH(req: Request, { params }: { params: Promise<{ id: string }> }) {
  const session = await auth()
  if (!session?.user?.id)
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 })

  const { id } = await params
  const { name, position, job, email, phone, residence } = await req.json()

  const employee = await prisma.employee.update({
    where: { id },
    data: {
      ...(name && { name }),
      email: email ?? null, position: position ?? null,
      job: job ?? null, phone: phone ?? null, residence: residence ?? null,
    },
  })

  if (name && employee.userId) {
    await prisma.user.update({ where: { id: employee.userId }, data: { name } })
  }

  return NextResponse.json(employee)
}

export async function DELETE(_req: Request, { params }: { params: Promise<{ id: string }> }) {
  const session = await auth()
  if (session?.user?.role !== "ADMIN")
    return NextResponse.json({ error: "Forbidden" }, { status: 403 })

  const { id } = await params
  const employee = await prisma.employee.findUnique({
    where: { id }, select: { userId: true },
  })

  if (employee?.userId) {
    await prisma.user.delete({ where: { id: employee.userId } })
  } else {
    await prisma.employee.delete({ where: { id } })
  }

  return new NextResponse(null, { status: 204 })
}

🗄️ DB 모델

TSprisma/schema.prisma — Employee
model Employee {
  id        String    @id @default(cuid())
  name      String
  email     String?
  phone     String?
  position  String?
  job       String?
  residence String?
  userId    String?  @unique
  user      User?   @relation(fields: [userId], references: [id], onDelete: Cascade)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
필드타입설명
nameString직원 이름 (필수)
emailString?이메일 (선택)
phoneString?전화번호 (선택)
positionString?직책 (선택)
jobString?업무 (선택)
residenceString?거주지 (선택)
userIdString? @uniqueUser와 1:1 연결 (선택 — User 없는 Employee 가능)

🔒 보안

  • ✓ PATCH — 로그인 여부만 검증 (session.user.id 확인), 401 반환
  • ✓ DELETE — ADMIN role 서버 검증 필수, 403 반환
  • ✓ 전화번호·거주지 등 개인정보는 로그인한 사용자에게만 노출 (proxy.ts 인증 게이트)
  • ✓ userId가 있는 Employee 삭제 시 User 삭제 → onDelete: Cascade로 Employee 자동 제거
  • ✓ userId가 없는 Employee는 prisma.employee.delete로 직접 삭제
