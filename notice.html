공지사항

사내 공지사항 CRUD 기능입니다. 공지 등록 시 전체 직원에게 웹 푸시 알림을 발송합니다.

마지막 업데이트 2026-05-14

📋 개요

  • → /notice — 목록 (최신순)
  • → /notice/new — 작성 (ADMIN)
  • → /notice/[id] — 상세
  • → /notice/[id]/edit — 수정 (ADMIN)
💡

공지 등록·수정·삭제는 ADMIN만 가능합니다. 등록 시 lib/push.ts를 통해 전체 직원에게 웹 푸시 알림을 발송합니다.

🌐 API 라우트

Method경로설명권한

POST/api/notices공지 생성 + 전원 웹 푸시 발송ADMIN

TSapp/api/notices/route.ts — POST
import { NextResponse } from "next/server"
import { auth } from "@/lib/auth"
import { db } from "@/lib/db"
import { sendPushToAll } from "@/lib/push"

export async function POST(req: Request) {
  const session = await auth()
  if (session?.user.role !== "ADMIN")
    return NextResponse.json({ error: "Forbidden" }, { status: 403 })

  const { title, content } = await req.json()

  const notice = await db.notice.create({
    data: { title, content, authorId: session.user.id },
  })

  // 전체 직원 웹 푸시 발송 (실패해도 공지는 저장됨)
  await sendPushToAll({
    title:   "새 공지사항",
    body:    notice.title,
    url:     `/notice/${notice.id}`,
  }).catch(console.error)

  return NextResponse.json(notice, { status: 201 })
}

🔔 웹 푸시 연동

공지 등록 시 lib/push.ts의 sendPushToAll()을 호출합니다. 내부적으로 DB의 PushSubscription 레코드를 조회해 모든 구독자에게 발송합니다.

⚠️

push 실패 처리: 푸시 발송 실패(410 Gone 등)는 무효 구독이므로 DB에서 삭제해야 합니다. sendPushToAll 내부에서 자동 처리되어야 합니다.
